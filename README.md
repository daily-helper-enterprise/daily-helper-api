| **US01** | Como desenvolvedor, quero registrar o que fiz ontem, o que farei hoje e meus impedimentos, para compartilhar meu progresso diário com o time. | - O usuário deve preencher três campos obrigatórios: “Ontem”, “Hoje” e “Impedimentos”.<br> - O registro deve ser salvo com data e associado ao usuário logado.<br> - O sistema deve impedir o envio se algum campo obrigatório estiver vazio.                            | M       |
| **US02** | Como Scrum Master, quero criar equipes e atribuir papéis aos membros para organizar o time.                                                   | - O Scrum Master deve poder cadastrar, editar e excluir equipes, definindo papéis (SM, PO, Dev).<br> - Apenas o Scrum Master pode realizar essas operações.                                                                                                              | L       |
| **US03** | Como membro da equipe, quero editar meu registro diário antes do horário limite, para corrigir informações caso algo mude.                    | - O registro só pode ser editado até o horário definido pelo Scrum Master.<br> - Após o limite, o botão “Editar” deve ficar desabilitado.<br> - O sistema deve manter o histórico de alterações.                                                                         | M       |
| **US04** | Como membro da equipe, quero visualizar minhas Dailies anteriores, para acompanhar minha evolução durante a sprint.                           | - Deve haver uma tela ou aba “Minhas Dailies” listando os registros anteriores.<br> - Os registros devem ser exibidos por ordem de data (mais recente primeiro).<br> - O usuário deve conseguir filtrar por intervalo de datas.                                          | S       |
| **US05** | Como Scrum Master, quero gerar relatórios diários com as respostas dos membros, para ter uma visão consolidada do andamento da equipe.        | - O relatório deve exibir todos os registros do dia agrupados por membro.<br> - Deve ser acessível em PDF e também visualizado na aplicação.<br> - Deve destacar visualmente os impedimentos abertos.                                                                    | M       |
| **US06** | Como Product Owner, quero gerar relatórios semanais com o resumo das Dailies, para acompanhar o progresso geral da sprint.                    | - O sistema deve permitir gerar o relatório semanal automaticamente ao fim da sprint ou sob demanda pelo PO.<br> - O relatório deve mostrar total de tarefas concluídas, pendentes e impedimentos.<br> - O relatório deve estar disponível para download e visualização. | L       |
| **US07** | Como Scrum Master, quero visualizar todos os impedimentos registrados na sprint, para priorizar e resolver os que mais impactam o time.       | - Deve haver uma tela “Impedimentos” listando todos os abertos.<br> - Cada impedimento deve conter autor, descrição e status (aberto/resolvido).<br> - O Scrum Master deve poder filtrar por usuário ou data.                                                            | M       |
| **US08** | Como Scrum Master, quero marcar um impedimento como resolvido, para manter atualizado o status da equipe.                                     | - Deve existir um botão “Marcar como resolvido”.<br> - O status deve mudar imediatamente e ser salvo no banco de dados.<br> - O histórico do impedimento deve permanecer acessível.                                                                                      | S       |
| **US09** | Como desenvolvedor, quero ser notificado quando meu impedimento for resolvido, para poder retomar minhas tarefas.                             | - O sistema deve enviar uma notificação (por e-mail ou no app) quando o status mudar, de acordo com a preferência do usuário.<br> - A notificação deve conter o nome do Scrum Master e o impedimento resolvido.                                                          | S       |
| **US10** | Como membro da equipe, quero realizar login na aplicação, para acessar minhas Dailies com segurança.                                          | - O login deve exigir e-mail e senha.<br> - As senhas devem ser armazenadas de forma criptografada.<br> - Logout deve encerrar a sessão.                                                                                                                                 | S       |
| **US11** | Como membro da equipe, quero visualizar as Dailies agrupadas por sprint, para acompanhar a evolução do time ao longo do ciclo.                | - Deve exibir lista de sprints com suas Dailies.<br> - Deve permitir filtrar por sprint.<br> - Deve mostrar resumo de progresso e impedimentos.                                                                                                                          | M       |
| **US12** | Como membro da equipe, quero receber lembretes antes do horário da Daily, para não esquecer de preencher meu registro.                        | - O sistema envia notificação push ou e-mail configurável pelo SM.<br> - O lembrete deve ser enviado 15 minutos antes do horário definido.                                                                                                                               | S       |
| **US13** | Como usuário, quero acessar o sistema em dispositivos móveis, para preencher minha Daily de forma prática de onde eu estiver.                 | - Interface deve ser responsiva.<br> - Formulários devem se ajustar corretamente a telas menores.                                                                                                                                                                        | M       |
| **US14** | Como Scrum Master, quero acessar o histórico de Dailies anteriores, para consultar dados de sprints passadas.                                 | - Deve listar Dailies antigas agrupadas por sprint.<br> - Deve permitir filtrar por data, equipe ou membro.                                                                                                                                                              | M       |
